# 校园OA系统产品需求文档

## 1. 产品概览
校园OA系统是一款为学校管理提供一站式解决方案的平台，旨在提高学校管理效率，优化师生沟通，实现数字化校园建设。
- 该系统主要解决学校日常办公、信息发布、流程审批等问题，为学校管理层、教师和学生提供便捷的信息化服务。
- 系统的目标是构建一个高效、安全、易用的校园管理平台，促进学校管理的规范化和现代化。

## 2. 核心功能

### 2.1 功能模块
我们的校园OA系统包含以下主要页面：
1. **登录页面**：用户认证入口，支持不同角色登录。
2. **管理员控制台**：系统管理、用户管理、权限管理等。
3. **教师工作台**：课程管理、学生管理、成绩管理等。
4. **学生工作台**：课程查询、成绩查询、请假申请等。
5. **通知公告**：校园新闻、活动通知等。
6. **流程审批**：请假、报销、采购等审批流程。

## 3. 核心流程

### 3.1 用户登录流程
1. 用户访问登录页面
2. 输入用户名和密码
3. 系统验证用户身份和角色
4. 根据角色跳转到对应的工作台

### 3.2 请假审批流程
1. 学生提交请假申请
2. 系统自动发送审批通知给辅导员
3. 辅导员审批（通过/拒绝）
4. 系统通知学生审批结果

### 3.3 成绩管理流程
1. 教师录入学生成绩
2. 系统自动计算总分和平均分
3. 学生查询个人成绩
4. 教师查看成绩统计分析

### 3.4 实习材料管理流程
1. 学生线上提交实习手册、实习鉴定表等材料
2. 班主任一键收集本班学生实习材料
3. 班主任审核材料，可催办、退回补充
4. 系统统计实习完成率、未提交名单

### 3.5 异常学生材料管理流程
1. 班主任线上记录、上传异常学生相关材料
2. 材料分类存储、快速检索
3. 提交至学生管理处审批
4. 流程可追溯，支持材料到期提醒

### 3.6 科技处课题申报材料评审流程
1. 科研教师线上提交课题申报书、佐证材料
2. 科技处组织线上评审，支持多人评审、评分统计
3. 评审结果公示、材料归档
4. 课题数据统计与报表生成

### 3.7 教研室课程标准评审流程
1. 教师线上提交课程标准初稿
2. 教研室组织线上评审，集体讨论、修改意见
3. 评审通过后提交教务处备案
4. 课程标准版本管理、评审意见汇总与跟踪

### 3.8 学生请假材料提交与审核流程
1. 学生线上填写请假申请，上传请假证明材料
2. 班主任线上审核请假申请，支持批准、驳回
3. 请假记录自动同步至学生考勤系统
4. 请假类型区分、时长自动计算、批量审批

### 3.9 学生学分申请与审批流程
1. 学生线上提交学分申请材料
2. 班主任收集本班学分申请材料并初审
3. 教务处终审，结果通知学生
4. 学分申请材料校验、数据自动汇总至教务系统

### 3.10 学生课程查询流程
1. 学生线上查询个人课表、课程介绍、上课地点、教师信息
2. 教师查询所授课程详情
3. 课程信息实时更新（如调课通知）
4. 课表导出、课程评价、历史课程查询

### 3.11 学生成绩查询流程
1. 学生线上查询个人期末成绩、平时成绩、补考成绩
2. 教师线上录入、修改成绩
3. 教务处审核成绩，支持成绩申诉通道
4. 成绩统计与排名、历史记录追溯、成绩单在线打印

## 4. 用户角色

| 角色 | 注册方式 | 核心权限 |
|-----|---------|--------|
| 管理员 | 系统预设 | 系统管理、用户管理、权限管理、所有功能访问权限 |
| 教师 | 管理员创建 | 课程管理、学生管理、成绩管理、请假审批、实习管理、异常学生管理、科研管理、课程标准管理 |
| 学生 | 管理员创建/批量导入 | 课程查询、成绩查询、请假申请、实习材料提交、学分申请 |

| 教务处人员 | 管理员创建 | 课程标准评审、学分申请终审、成绩审核 |
| 学生管理处人员 | 管理员创建 | 异常学生材料审核、材料跟踪处理 |
| 班主任 | 管理员创建 | 学生实习材料收集、异常学生管理、请假审批、学分申请初审 |

## 5. 系统需求

### 5.1 技术栈
- 后端：Java Spring Boot + MyBatis
- 前端：Vue
- 数据库：MySQL
- 认证：JWT

### 5.2 非功能需求
- 安全性：数据加密、权限控制、防止SQL注入等
- 性能：响应时间快、支持并发访问
- 可靠性：系统稳定运行、数据备份与恢复
- 易用性：界面友好、操作简单
- 可扩展性：支持后续功能模块的添加和集成

## 6. 数据需求

### 6.1 核心数据实体
- 用户（User）：包含管理员、教师、学生、科技处人员、教务处人员、学生管理处人员
- 角色（Role）：用户角色定义
- 权限（Permission）：系统功能权限
- 课程（Course）：课程信息
- 班级（Class）：班级信息
- 学生（Student）：学生详细信息
- 教师（Teacher）：教师详细信息
- 成绩（Score）：学生成绩
- 请假（Leave）：请假申请
- 公告（Announcement）：校园公告
- 审批流程（Approval）：各类审批流程
- 实习材料（InternshipMaterial）：学生实习相关材料
- 异常学生材料（AbnormalStudentMaterial）：异常学生相关材料
- 课题申报（ProjectApplication）：科研课题申报信息
- 课程标准（CourseStandard）：课程标准信息
- 学分申请（CreditApplication）：学生学分申请信息
- 材料（Material）：通用材料存储
